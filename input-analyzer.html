Dataset key detail

Each row is a dataset key (the key in monitoring.dataset in the diagnostic, shown here as a column) — not the same as the ingestion buffer (FB_ACTIVE) in Event flow. Each value is from the latest 30 s row in D-input-processing.tsv for that key. The underlying JSON path is monitoring.dataset.<dataset_key>.processing_time.histogram (extractor converts ns → ms). The histogram is a lifetime reservoir, not a per-window delta: mean / median / percentiles reflect how long codec + input-side processing took for observations accumulated in the reservoir for that key since it started.
